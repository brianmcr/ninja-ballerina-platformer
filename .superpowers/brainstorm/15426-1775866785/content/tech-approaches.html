Tech Approach for Ninja Ballerina

Pick the engine/framework that fits best

A

Vanilla Canvas + TypeScript

Hand-roll everything — game loop, physics, rendering, sprite system. Maximum learning, maximum control, but slow to get to "fun."

Pros

  • Zero dependencies
  • Learn how engines work
  • Total control

Cons

  • Weeks before it feels like a game
  • Camera, collision, tilemaps all from scratch
  • Sprite animation system is non-trivial
B

Kaplay (formerly Kaboom.js)

Lightweight 2D game library designed for platformers. Handles physics, sprites, scenes, tilemaps. Still browser-based, still TypeScript. Recommended.

Pros

  • Built for exactly this kind of game
  • Platformer physics out of the box
  • Tilemap + sprite + scene management
  • Active community, good docs
  • ~15KB gzipped

Cons

  • Less "from scratch" learning
  • API opinions you might disagree with
C

Phaser 3

The heavyweight champion of browser 2D engines. Massive feature set — physics (Arcade + Matter.js), tilemaps, particles, tweens, audio, input.

Pros

  • Battle-tested, huge ecosystem
  • Tiled map editor integration
  • Every feature you could want

Cons

  • ~1MB bundle, heavier API surface
  • More complex setup
  • Overkill for a single-world platformer
